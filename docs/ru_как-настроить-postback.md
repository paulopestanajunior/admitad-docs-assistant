# Как настроить Postback

Если вы используете собственную систему статистики продаж и лидов, вы можете добавить в [Affilead](https://affilead.ru/?utm_source=support.admitad.ru&utm_medium=referral ) postback URL вашего сервера, чтобы автоматически получать данные о конверсиях.


У Affilead есть инструмент Postback, который позволяет получать эти данные и использовать их для аналитики и оптимизации работы.


Данные передаются с сервера Affilead на ваш сервер через GET или POST HTTP\-запросы. Далее вы можете преобразовывать их в таблицы, графики и диаграммы или использовать в собственных приложениях. Вся статистика собирается в одном месте, что позволяет вам эффективно контролировать и анализировать данные сразу по нескольким партнерским сетям.

# Как добавить Postback


1\. Перейдите в «Инструменты» и в блоке Postback нажмите «Подробнее».


2\. Нажмите «Добавить Postback».


Вы можете добавить несколько postback URL, настроив их под различные нужды.


3\. Выберите площадку и оффер, по которым хотите получать статистику о действиях


4\. В поле «Тип запроса» выберите метод получения информации, который вам удобен.


Основное различие методов GET и POST заключается в способе передачи данных:

- метод GET отправляет вам всю собранную информацию как часть URL, добавляя параметры и их значения в хвост ссылки;
- метод POST передает данные в теле запроса, а не в самом URL.


Подробнее об особенностях каждого запроса вы можете прочитать [здесь](http://www.w3schools.com/tags/ref_httpmethods.asp).


 5\. В поле «Статус действия» выберите статусы действий, по которым хотите получать данные.

- Открыто — статус появляется, когда посетитель, которого вы привели на сайт бренда, совершил целевое действие, и оно впервые попало в статистику. Действие будет в этом статусе, пока бренд не подтвердит или отклонит его.
- Подтверждено — бренд проверил действие и подтвердил его.
- Отклонено — бренд проверил действие и отклонил его.


6\. Укажите ссылку на ваш сервер, по которой будет приходить HTTP\-запрос.


7\. Выберите параметры, которые будут включены в URL. Подробнее [Типы параметров и их допустимые значения](#param)


8\. Нажмите «Сохранить».


9\. Нажмите «Тестировать код», чтобы проверить, что нужные данные поступают на ваш сервер.


Список всех ваших Postback URLs находится на странице [Postback](https://app.affilead.ru/tools/postbacks)

## Типы параметров и их допустимые значения

| Имя параметра | Описание |
| --- | --- |
| offer\_id | ID оффера |
| offer\_name | Название оффера |
| source\_id | ID площадки |
| source\_name | Название площадки |
| order\_id | ID заявки |
| status | Статус действия |
| title | Название действия или тарифа |
| subid1 | SubID1 |
| subid2 | SubID2 |
| subid3 | SubID3 |
| action\_time | Время действия (время, когда действие пришло с постбеком от бренда и было зафиксировано в системе Affilead) |
| amount | Сумма комиссии |
| click\_time | Время клика (время фактического совершения действия) |
| country\_code | Код страны |
| currency | Валюта |
| erid | Идентификатор рекламы |
| user\_agent | User\-Agent |
| user\_referer | User\-Refferer |
| uuid | ID действия |
| rate\_id | Информация о тарифе |
