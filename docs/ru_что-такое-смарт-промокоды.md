# Что такое смарт-промокоды

Смарт\-промокоды позволяют веб\-мастерам продвигать ваши предложения, а также отслеживать действия пользователей. Они могут использоваться как с партнерскими ссылками, так и без них.


Совместное использование трекинга по партнерской ссылке и промокоду обеспечивает более надежное отслеживание заказов, даже с учетом ограничений браузеров и блокировщиков рекламы. Такой подход защищает веб\-мастеров от потери заказов и повышает точность атрибуции.

Преимущества совместного трекинга по ссылке и промокоду:

- Надежность отслеживания. Если трекинг по партнерской ссылке не сработает, заказ будет отслежен по промокоду, и наоборот, что минимизирует риск потери заказов.
- Управление доступом. Возможность выдавать промокод определенной площадке и модерировать доступ к каждому промокоду.
- Перераспределение промокодов. Промокод можно отозвать, если веб\-мастер использует его неэффективно, и передать другой площадке.
- Быстрое обновление статистики. Использование трекинга по ссылке ускоряет отображение заказов в статистике.
- Разметка трафика. Дает возможность веб\-мастерам размечать источники трафика, заказы и аудиторию.


Читайте также [Как добавить смарт\-промокод в партнерскую программу](https://support.admitad.ru/knowledge-base/article/%D0%BA%D0%B0%D0%BA-%D0%B4%D0%BE%D0%B1%D0%B0%D0%B2%D0%B8%D1%82%D1%8C-%D1%81%D0%BC%D0%B0%D1%80%D1%82-%D0%BF%D1%80%D0%BE%D0%BC%D0%BE%D0%BA%D0%BE%D0%B4%D1%8B-%D0%B2-%D0%BF%D0%B0%D1%80%D1%82%D0%BD%D0%B5%D1%80%D1%81%D0%BA%D1%83%D1%8E-%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D1%83)

## Способы интеграции

# 


Для отслеживания действий по смарт\-промокодам вы можете использовать:

- Онлайн\-интеграцию, которая включает:
	- [бэкенд\-интеграцию через postback\-запрос, XML или API](#back-integration);
	- [фронтенд\-интеграцию через TagTag, GTM Template](#front-integration).
- [Оффлайн\-интеграцию](#off-integration) ー трекинг через ручную загрузку файла в систему.


Подробнее о способах интеграции в статье [Интеграция: что это такое и как выбрать способ интеграции с Admitad](https://support.admitad.ru/knowledge-base/article/%D0%B8%D0%BD%D1%82%D0%B5%D0%B3%D1%80%D0%B0%D1%86%D0%B8%D1%8F-%D1%87%D1%82%D0%BE-%D1%8D%D1%82%D0%BE-%D1%82%D0%B0%D0%BA%D0%BE%D0%B5-%D0%B8-%D0%BA%D0%B0%D0%BA-%D0%B2%D1%8B%D0%B1%D1%80%D0%B0%D1%82%D1%8C-%D1%81%D0%BF%D0%BE%D1%81%D0%BE%D0%B1-%D0%B8%D0%BD%D1%82%D0%B5%D0%B3%D1%80%D0%B0%D1%86%D0%B8%D0%B8-%D1%81-admitad_1)


Наиболее эффективной и предпочтительной является онлайн\-интеграция с трекингом как по ссылке, так и по промокоду. Она ускоряет появление данных в статистике, что дает возможность веб\-мастерам оперативно анализировать источники трафика и оптимизировать их. При необходимости вы также можете дополнять статистику данными через ручную загрузку файла в систему.


Если вы используете онлайн\-интеграцию только для трекинга по ссылке или она полностью невозможна для вас, для трекинга по промокоду можно использовать оффлайн\-интеграцию. Однако у этого способа есть недостатки:

- Веб\-мастера не смогут разметить трафик и корректно атрибутировать заказы, что особенно критично для кешбэк\-сервисов.
- Данные о применении промокодов будут доступны только после загрузки файла в систему, что замедляет анализ и оптимизацию трафика для веб\-мастеров.

### Бэкенд\-интеграция через postback\-запрос, XML или API


Бэкенд\-интеграция предполагает передачу данных о действиях с сервера. Этот способ обеспечивает точное отслеживание действий, независимо от политики конфиденциальности браузеров, блокировщиков рекламы или других ограничений.


Если у вас уже настроена данная интеграция для отслеживания действий по партнерским ссылкам, для работы со смарт\-промокодами вам нужно дополнить ее настройки.

### Фронтенд\-интеграция через TagTag, GTM Template


При фронтенд\-интеграции данные о действиях передаются с сайта через браузер пользователя. Политики конфиденциальности и блокировщики рекламы существенно влияют на этот способ, поэтому его не стоит использовать как единственный метод отслеживания.


Для точного отслеживания необходимо также предоставлять файл с данными о действиях по промокодам в дополнение к фронтенд\-интеграции. Такой подход обеспечит учёт всех действий.


При настройке фронтенд\-интеграции через TagTag или Google Tag Manager (GTM) убедитесь, что код работает на каждой странице без каких\-либо условий: например, запуск только для трафика Admitad.


Также необходимо добавить скрипты и cookies Admitad в исключения системы контроля контента или политики безопасности контента и пометить их как Strictly Necessary.

### Оффлайн\-интеграция


При оффлайн\-интеграции вы самостоятельно передаёте данные о действиях в Admitad для их обработки и учёта.


Для точности отслеживания важно использовать полный и достоверный источник данных, такой как ваша CRM\-система или база данных. Эти системы обеспечивают точность информации о действиях пользователей.


Google Analytics 4 и аналогичные инструменты не подходят в качестве источника данных для оффлайн\-интеграции из\-за технических ограничений: их код отслеживания часто блокируется браузерами, что может привести к потере данных о действиях.


Каждую неделю вам нужно экспортировать статистику по действиям из вашей системы и передавать файл менеджеру для загрузки данных в ваш личный кабинет. Файл должен содержать следующую информацию:

- ID заказа;
- промокод;
- стоимость товара;
- дата действия.

## Правила атрибуции

- Действия с использованием смарт\-промокодов Admitad всегда засчитываются в его пользу, даже если до применения промокода использовались другие источники отслеживания (например, другие партнерские ссылки). Все подтвержденные заказы по смарт\-промокодам должны быть оплачены.
- Необходимо отслеживать действия по смарт\-промокодам на всех устройствах. У мобильных устройств могут быть ограничения на отслеживание, поэтому для них рекомендуем использовать оффлайн\-интеграцию.

## Требования к смарт\-промокодам

- При создании смарт\-промокодов используйте формат названий: Admitad1, Admitad2, Admitad3\. Это позволяет идентифицировать их как часть сети Admitad и отличать от общих предложений.
Если вы осуществляете деятельность на территории РФ, используйте формат названий: Starsi1, Starsi2, Starsi3\.  
- Для повышения эффективности и стабильности программы рекомендуем использовать бессрочные смарт\-промокоды. Вам не придется следить за их сроком действия и продлять его, что исключит потерю трафика и заказов.
Если бессрочные промокоды невозможны, то устанавливайте срок действия не менее 90 дней.
- Смарт\-промокоды должны распространяться на широкий ассортимент товаров, чтобы привлечь больше пользователей и повысить интерес веб\-мастеров. Ограничение количества товаров может снизить эффективность промокодов.
- Скидка по промокодам Admitad должна быть выше, чем общедоступная скидка.
