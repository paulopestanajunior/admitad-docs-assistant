# What advertisers pay for

Admitad Partner Network is a CPA network where advertisers pay for [actions](https://support.admitad.com/hc/en-us/articles/4403304880529#action-en).


That means that an advertiser pays a reward to a publisher each time when a user follows the publisher's affiliate link and performs an action on the advertiser's website.


A target action can be an act of order placement, registration on a website, app install, application, etc. [Learn more about types of target actions](#actions)


A program may support several target actions, each having an individual rate and model. Also, one action may have different rates for different conditions. [Learn more about rates](#rates)

### Paid order


The most frequent action type. According to its terms, you need to bring a user who will buy a product or a service. If a user:

- placed but did not pay for an order,
- did not receive the order or canceled it,
- returned the order,


it is assumed that the action is not completed, so the advertiser will decline it.  
[More about reasons for declining actions](https://support.admitad.com/hc/en-us/articles/360019116357)

### Confirmed order


If a user ordered a product and confirmed the order by phone, an advertiser will pay you reward.

### Sign Up


Depending on a program category a user should fill a registration form on a website, in an online game, or in a mobile app.

### Confirmed registration


A user registered on a website, in a game, or in an app, and confirmed their registration via email.

### Effective registration


Also known as quality registration, it is used only in online games. A broad term that includes anything that is more complicated than registration but does not require much effort from a user, for example, achieving level 1\. The set of efforts varies depending on the program and is not disclosed to avoid fraud.

### Completed application


Is used mainly by financial programs: a user needs to apply for credit, insurance, credit card, etc.

### Confirmed application


A user filled out an application and the application was confirmed by the call center of the advertiser. Depending on the program category, a user can apply for a flower delivery (in the online services), for a credit card or a loan (in the financial programs), for a tour (in the travel programs).

### Approved application


The user has submitted an application and the advertiser approved it.


Confirmed application is not the same as approved application. In the first case, the advertiser just confirms that the application was received. In the second case, the advertiser takes a positive decision on the application.

### Loan granted, card issued


The user has taken a loan or gotten a card.

### Install


A user should install a mobile application and necessarily open it, otherwise, the action will not be tracked by the system. This type of action is used only in mobile programs.

### Active player


This action is used only in online games. You get rewarded for users who registered and became active players in an advertiser’s game.  
An active player is a user who regularly plays the game. The parameters that define a user as active are not disclosed to avoid fraud.

### Subscription


It is used only in mobile programs. To get rewarded, you should encourage a user to subscribe to use the app.

### Payment, set of access rights


Only used in online games. Means purchasing access to a game or any other in\-app purchase.

## Rates


Some advertisers add certain actions instead of separate rates. In this case, you will see several actions with one or several rates specified for each of them.


For each action, the advertiser specifies a rate—i.e. a bonus amount.  
For one action, an advertiser may set different rates for the following:

- [different categories of products or services](#categories),
- [existing and new customers](#new-old-customer),
- [use or non\-use of coupons or promo codes](#coupon),
- [different countries](#countries),
- [different order amounts](#order-amount).


For that, an advertiser can create several rates for one action.


If the rate is specified as a range in the program card, it means the program has several target actions or several rates are added for one action.

  
Click on the rate value to view for which the advertiser pays, and how much.

### Product categories


In the action name, an advertiser can specify a category name ("Household appliances", "Mobile phones") or its number (Category 1, Category 2, etc.). In the second case, the explanation of categories is given in a program description.

### Existing or new customers

- A new customer (new client, new user) is a user who places an order on the website for the first time.
- An existing customer (existing client, existing user) is a user who ordered on this website or app before.


This kind of rate is most often used in actions like Paid order. The rate for a new customer is usually higher than the rate for an existing one, as the advertisers often use an affiliate program to attract a new audience to the website.


If a program has only one of the rates, it means that the advertiser pays only for the first orders, or for repeated orders.

### Use or non\-use of coupons or promo codes


In this case, the reward depends on whether the user used a promo code or a coupon for a discount when placing an order. As a rule, if a coupon was applied, the reward for the order will be lower, because it is less profitable for the advertiser, and it is easier to attract the clients by promising discounts.

### Countries of customers


This rate is used mainly in mobile programs and online games. The reward depends on the country in which a user who performs an action is located.

### Order amount

In this case, the reward depends on the amount user pays for identical products (with the same SKUs).

- If the order contains a single product, for instance, an online learning course or loan, the reward is calculated based on the full order amount.
- If the order contains several different products, the reward is calculated based on the cost of each item.

The reward for different order amounts is possible in programs of certain categories if the order amount is passed as a single action, for instance, when one buys a set of services.

  

## How to find out what actions an advertiser pays for


To find out what the advertiser pays for, click on the commission value in the program card in [Admitad Store](https://store.admitad.com/en/catalog/). If the commission is specified as a range, it means the program has several target actions or rates.


In the new window, you will see for which target actions the advertiser pays, and how much:


Also, you can find information about actions the advertiser pays for on the affiliate program page, in the CPA Terms and CPC Terms sections. If a commission is specified as a range, it means there are several rates for the action. Click on the range to view them.

## Can the rate change?


Each target action has its own reward rate. In any affiliate program, the rate can change.   
If the rate changes in a program your ad space is connected to, we will notify you in advance: before the new rate takes effect.


The rate may also temporarily increase as part of incentive deals. During this period, the program will display the label “Increased rate”, and the deal details will be shown on the program page. To receive the increased rate, click Accept on the advertiser’s offer in your account. Acceptance means you commit to increasing traffic during the period of the increased rate. If you do not accept the offer, your reward will still be calculated at the default rate. 


If an action is created in the Admitad system before you accept the offer for the temporary rate increase, the reward will be credited at the rate that is set by default.
