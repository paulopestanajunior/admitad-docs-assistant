# Как выдать бухгалтеру доступ к выводу денег

Если аккаунт использует несколько сотрудников компании, мы рекомендуем, чтобы доступ к аутентификатору был только у владельца аккаунта.


Двухфакторная аутентификация нужна для подтверждения платежных данных — тех реквизитов, по которым вы будете получать выплаты. Для создания заявки на выплату двухфакторная аутентификация не требуется. Вы можете добавить и подтвердить платежные данные самостоятельно, а ваш бухгалтер сможет создавать заявки на вывод денег. 


Чтобы ограничить действия, которые сотрудники вашей компании могут выполнять в Wallet, вы можете создать для них гостевые аккаунты. 

В гостевом аккаунте нельзя настроить двухфакторную аутентификацию.


Чтобы создать гостевой аккаунт, обратитесь к вашему персональному менеджеру или напишите нам в [службу поддержки](https://support.admitad.ru/knowledge-base/article/%D0%BA%D0%B0%D0%BA-%D1%81%D0%B2%D1%8F%D0%B7%D0%B0%D1%82%D1%8C%D1%81%D1%8F-%D1%81-%D0%BF%D0%BE%D0%B4%D0%B4%D0%B5%D1%80%D0%B6%D0%BA%D0%BE%D0%B9-admitad_1). В запросе напишите:

1. Email сотрудника, на который нужно зарегистрировать гостевой аккаунт.
2. Какие права доступа нужно предоставить гостевому аккаунту.  
Возможные права:
	- Добавление налоговых данных
	- Изменение налоговых данных
	- Удаление налоговых данных
	- Добавление платежных данных
	- Изменение платежных данных
	- Удаление платежных данных
	- Вывод денежных средств
	- Просмотр истории выплат
3. Должность сотрудника, для которого создается аккаунт (не обязательно).


При создании гостевого аккаунта компании также создается отдельный аккаунт сотрудника в Wallet, куда он сможет заходить под паролем, известным только ему. Этот аккаунт привязан к основному аккаунту компании в качестве гостевого.


Когда сотрудник войдет в свой аккаунт в Wallet (это можно сделать по [прямой ссылке](https://earnings.admitad.com)), он увидит модальное окно, в котором сможет переключиться на гостевой аккаунт вашей компании. В нем сотруднику будут доступны только те действия, которые вы выбрали (см. список прав доступа выше). 


Например, вы предоставили гостевому аккаунту права «Вывод денежных средств» и «Просмотр истории выплат». Ваш сотрудник не сможет добавлять и редактировать налоговую информацию и платежные данные, но сможет создавать заявки на вывод денег на те платежные реквизиты, которые вы добавили и подтвердили сами. Также он сможет просматривать раздел «История операций».
