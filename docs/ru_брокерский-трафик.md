# Брокерский трафик

Брокерский трафик — вид [трафика](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#traffic-en), который используется только в финансовых [программах](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#affiliate-program-en).


В таких программах [рекламодателем](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#advertiser-en) обычно является банк или другая финансовая организация, которая хочет получать заявки пользователей на финансовые услуги: оформление кредита или карты, открытие счета и т.д.


Веб\-мастер собирает заявки пользователей на такие услуги  и отправляет их рекламодателю. Тем самым он выступает в качестве брокера — посредника между рекламодателем и пользователем.


При работе с брокерским трафиком веб\-мастер приводит пользователя на свою [площадку](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#ad-space-en). Там пользователь должен заполнить и отправить заявку на получение финансовой услуги. Собранные заявки веб\-мастер передает рекламодателю через [Admitad API](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#api-en).


Поскольку веб\-мастеру не нужно вести пользователя на сайт рекламодателя, [партнерские ссылки](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#affiliate-link-en) при работе с брокерским трафиком не используются.  
 

## Схема работы


1\. Вы создаете [лендинг](https://support.admitad.ru/knowledge-base/article/%D0%B3%D0%BB%D0%BE%D1%81%D1%81%D0%B0%D1%80%D0%B8%D0%B9-admitad-partner-network_1#landing-en) и размещаете на нем форму заявки на финансовую услугу.  
Список обязательных полей в заявке определяется программой ([см. ниже](#landing-page)).


2\. Пользователь переходит на лендинг, вносит свои данные в форму и отправляет заявку.


3\. С вашей площадки заявка с данными пользователя отправляется через API на сервер Admitad.


4\. Заявки проверяются на стороне Admitad, после чего передаются на сервер рекламодателя.


5\. Рекламодатель получает заявку и проверяет ее. Если заявка соответствует требованиям рекламодателя и правилам программы, он подтвердит заявку, и вы получите вознаграждение.  
 

## Требования к площадке

- В качестве площадки нужно использовать лендинг или веб\-приложение.
- У каждого пользователя нужно запрашивать разрешение на передачу и обработку персональных данных, после того как он заполнит форму заявки.
- На сервере должен быть установлен валидный SSL\-сертификат.
- Передавать данные пользователя на сервер рекламодателя можно только по защищенному протоколу https.

### 1\. Выберите программу, которая работает с брокерским трафиком

#### Способ 1: получите список программ через API


Чтобы получить список программ, которые работают с брокерским трафиком, отправьте запрос с параметром has\_tool\=brocker\_traffic с помощью [этого метода API](https://developers.admitad.com/ru/knowledge-base/article/%D0%BF%D0%B0%D1%80%D1%82%D0%BD%D0%B5%D1%80%D1%81%D0%BA%D0%B8%D0%B5-%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D1%8B). 


Выберите программу из полученного списка и найдите ее в [Admitad Store](https://store.admitad.com/ru/catalog/). Откройте страницу этой программы.


После этого перейдите к следующему шагу — [подключению к программе](#joining-program).  
 

#### Способ 2: выберите программу вручную


Откройте [Admitad Store](https://store.admitad.com/ru/catalog/) и в фильтре по виду трафика выберите «Брокерский трафик».


Брокерский трафик в найденных программах может быть разрешен полностью или только по согласованию с рекламодателем. Чтобы проверить, какие условия работы с брокерским трафиком действуют в программе, перейдите на ее страницу. В меню слева разверните пункт «Правила» и перейдите в раздел «Виды трафика».


В таблице с видами трафика найдите «Брокерский трафик», напротив него вы увидите условие:

- Можно — означает, что в этой программе брокерский трафик разрешен, и после подключения к программе вы можете начать работу с ним.
- По согласованию — означает, что в этой программе возможность работать с брокерским трафиком нужно сначала согласовать с рекламодателем. Для этого [обратитесь в поддержку Admitad](https://support.admitad.ru/knowledge-base/article/%D0%BA%D0%B0%D0%BA-%D1%81%D0%B2%D1%8F%D0%B7%D0%B0%D1%82%D1%8C%D1%81%D1%8F-%D1%81-%D0%BF%D0%BE%D0%B4%D0%B4%D0%B5%D1%80%D0%B6%D0%BA%D0%BE%D0%B9-admitad_1).

### 2\. Подключитесь к программе


Для этого на странице партнерской программы нажмите кнопку «Подключиться» или «Подать заявку».


Кнопка «Подать заявку» значит, что в программе предусмотрена модерация.  
Нажмите на нее, чтобы отправить запрос на подключение к программе. Примерное время модерации указано слева от кнопки. Вы сможете работать с программой, только после того как рекламодатель одобрит вашу площадку.  
[Подробнее о том, как подключаться к программам](https://support.admitad.ru/knowledge-base/article/%D0%BA%D0%B0%D0%BA-%D0%BF%D0%BE%D0%B4%D0%BA%D0%BB%D1%8E%D1%87%D0%B8%D1%82%D1%8C%D1%81%D1%8F-%D0%BA-%D0%BF%D0%B0%D1%80%D1%82%D0%BD%D0%B5%D1%80%D1%81%D0%BA%D0%BE%D0%B9-%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B5_1)

### 3\. Настройте авторизацию Admitad API


Брокерские заявки передаются через Admitad API.


Для работы с API вам нужно пройти авторизацию и получить access\_token.


Подробнее о процессе читайте в [документации API](https://developers.admitad.com/ru/knowledge-base/article/%D0%BA%D0%BB%D0%B8%D0%B5%D0%BD%D1%82%D1%81%D0%BA%D0%B0%D1%8F-%D0%B0%D0%B2%D1%82%D0%BE%D1%80%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F).


В параметре scope указываются методы Admitad API, к которым будет открыт доступ. Вы можете получить доступ к любому методу, но для этого укажите их вручную в параметре scope.


Пример значения параметра scope для работы с брокерским трафиком:

```
scope=advcampaigns broker_application manage_broker_application

```

Со справкой по настройкам доступа вы можете ознакомиться [здесь](https://developers.admitad.com/ru/knowledge-base/article/%D1%81%D0%BF%D1%80%D0%B0%D0%B2%D0%BA%D0%B0-%D0%BF%D0%BE-%D0%BD%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B9%D0%BA%D0%B0%D0%BC-%D0%B4%D0%BE%D1%81%D1%82%D1%83%D0%BF%D0%B0_1?search=89fc8e3e2808d8fc4266915c8917db12).


Срок действия access\_token составляет 1 неделю. Поэтому нужно реализовать его обновление с помощью [этого метода](https://developers.admitad.com/ru/knowledge-base/article/%D1%81%D0%BF%D1%80%D0%B0%D0%B2%D0%BA%D0%B0-%D0%BF%D0%BE-%D0%BD%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B9%D0%BA%D0%B0%D0%BC-%D0%B4%D0%BE%D1%81%D1%82%D1%83%D0%BF%D0%B0_1).

### 4\. Подготовьте лендинг


1\. Убедитесь, что ваша площадка соответствует [всем требованиями](#ad-space-requirements).


2\. На лендинге разместите форму заявки с обязательными полями. Список этих полей можно найти на странице программы (вкладка «Брокерский трафик») или в [документации](https://developers.admitad.com/ru/knowledge-base/article/%D0%B7%D0%B0%D1%8F%D0%B2%D0%BA%D0%B8-%D0%B1%D1%80%D0%BE%D0%BA%D0%B5%D1%80%D0%B0).  
Заявка не должна отправляться, если хотя бы одно из полей не заполнено. При попытке отправить такую заявку пользователь должен увидеть ошибку.


3\. Под формой нужно разместить чекбокс, в котором пользователь должен поставить галочку, чтобы дать согласие на передачу и обработку персональных данных.  
По умолчанию галочка в чекбоксе должна отсутствовать.


4\. Добавьте кнопку для отправки заполненной заявки: например, «Заказать», «Получить кредит» или «Отправить».  
 

### 5\. Настройте отправку заявок через Admitad API


Перед настройкой ознакомьтесь и примите правила работы с брокерским трафиком.  
Для этого перейдите на вкладку «Брокерский трафик» на странице программы, перейдите по ссылке с правилами, прочтите их и нажмите «Принять».


Метод отправки заявок описан в [документации](https://developers.admitad.com/ru/knowledge-base/article/%D0%B7%D0%B0%D1%8F%D0%B2%D0%BA%D0%B8-%D0%B1%D1%80%D0%BE%D0%BA%D0%B5%D1%80%D0%B0#creating-broker-applications).


Здесь можно указать, каким именно партнерским программам вы хотите отправлять заявки, а также задать алгоритм отправки заявок. 

### 6\. Настройте проверку статусов отправленных заявок


Актуальные статусы отправленных заявок проверяются [этим методом API](https://developers.admitad.com/ru/knowledge-base/article/%D0%B7%D0%B0%D1%8F%D0%B2%D0%BA%D0%B8-%D0%B1%D1%80%D0%BE%D0%BA%D0%B5%D1%80%D0%B0#list-of-applications-parameters).


После того как вы отправили заявку в партнерскую программу, заявка получит статус *processing*. Этот статус означает, что заявка с данными пользователя проходит проверку на стороне Admitad и ждет очереди на отправку рекламодателю.


Когда рекламодатель принимает заявку, заявке присваивается статус *waiting*. В статистике она появится в статусе «Ожидает».


Далее рекламодатель обрабатывает заявку:

- или подтверждает ее, и тогда заявка получает статус *approved*.
- или отклоняет, в этом случае заявка получает статус *declined*.

## Как брокерские заявки отображаются в статистике


Заявки через брокерский трафик фиксируются в статистике так же, как и обычные действия по партнерским ссылкам.

Особенности:

- В статистику Admitad попадают брокерские заявки, которые прошли проверку Admitad и успешно получены рекламодателем.
- Кликом считается факт получения брокерской заявки сервером Admitad.
- Время клика — момент времени, когда сервер Admitad получил заявку.
- Действием считается факт регистрации заявки рекламодателем.
- Время действия — время получения заявки рекламодателем.

## Возможные ошибки при работе с брокерским трафиком

| Текст ошибки | Пояснение |
| --- | --- |
| Кампания не найдена. | Данная партнерская программа не представлена в Admitad или приостановила свою работу. |
| Кампания не принимает заявки брокеров. | Данная программа не работает с брокерским трафиком. |
| Вы не подключены к данной кампании. | Чтобы работать с программой, сначала подключитесь к ней. [Как подключиться](#work-process), читайте выше. |
| Вам не разрешен брокерский трафик. | Рекламодатель не подтвердил вашу заявку на работу с брокерского трафика. |
| Вы превысили ограничение по количеству заявок. | В данной программе установлен лимит на количество заявок в день. Когда количество заявок достигает лимита, вы не можете передавать новые заявки до конца текущего дня. |
| Вам необходимо принять Соглашение о работе с брокерским трафиком. | Чтобы работать с брокерским трафиком, [ознакомьтесь с правилами работы и примите их](#sending-api-requests). |
| Заявки, в которых `user_notified = 0`, не принимаются в обработку. | Значение «0» в параметре `user_notified` означает, что пользователь не дал согласие на сбор и обработку его персональных данных. Такие заявки не передаются рекламодателю. |
| Кампания требует верификацию через SMS. | По правилам данной программы требуется, чтобы пользователь подтвердил заявку по SMS. [Как настроить SMS\-верификацию](https://developers.admitad.com/ru/knowledge-base/article/%D0%B7%D0%B0%D1%8F%D0%B2%D0%BA%D0%B8-%D0%B1%D1%80%D0%BE%D0%BA%D0%B5%D1%80%D0%B0#creating-api-applications) |
| Кампания не требует верификацию через SMS. | В данной программе не нужно запрашивать у пользователя подтверждение заявки по SMS. |
